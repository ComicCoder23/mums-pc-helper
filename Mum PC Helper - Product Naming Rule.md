MUM PC HELPER — PRODUCT NAMING RULE


Purpose
This document locks the naming rule for the product so the release does not drift between names.


CANONICAL PRODUCT NAME
Mum PC Helper


RULE
Use “Mum PC Helper” as the public-facing product name.
Do not rename the product to PC Helper for v1.


HOW TO USE THE NAME
Brand / product name:
- Mum PC Helper


Broader positioning can be expressed in subtitle or description, for example:
- Mum PC Helper — a safe first-pass Windows support kit
- Mum PC Helper — simple help for slow, cluttered, or confusing PCs
- Mum PC Helper — easy support for home and family Windows PCs


WHY THIS NAME STAYS
- simple
- warm
- memorable
- easy to understand quickly
- friendlier than a cold generic utility name


POSITIONING RULE
The name can stay Mum PC Helper even if the audience broadens.
The broader audience should be handled in:
- subtitle
- description
- user guide language
- product positioning copy


DO NOT DO
- do not switch between Mum PC Helper and PC Helper randomly
- do not create a second competing public product name for v1
- do not use one name in Drive, another in GitHub, and another in docs without reason


SYSTEM RULE
- Mum PC Helper = canonical product/brand name
- GitHub can still use a practical repository name if needed
- Drive customer-facing pack should use Mum PC Helper naming
- Notion release/control docs should reference Mum PC Helper as the canonical v1 product name


NEXT ACTION
Use this naming rule when doing the rename and packaging pass.


FINAL RULE
Lock the name.
Ship the product.
Do not reopen naming drift during v1.
